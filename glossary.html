메서드 오버라이드(Method Override) : 자식 클래스에서 부모 클래스의 매서드를 재정의 하는 것을 의미함.
이는 다형성을 구현하는 중요한 기법으로, 부모 클레스의 기본 동작을 자식 클래스에서 변경하고자 할 때 사용

인스턴스(Instance): 객체 지향 프로그래밍의 핵심으로, 클래스를 기반으로 만들어진 실제 객체를 의미한다. 인스턴스는 프로그램에서 실제로 동작하는 객체이며, 개발자는 인스턴스를 통해 클래스의 정의를 실질적으로 구현하고 활용한다. 코드의 재사용성과 확장성을 높이는 데 필수적인 요소이다. 예시

인스턴스의 생성: 클래스의 생성자를 호출하여 인스턴스를 생성한다. ex) new Car( Black , Kia );

인스턴스의 속성: 인스턴스는 클래스에서 정의한 속성을 자신의 고유한 값으로 가진다.
예시에선 myCar와 yourCar라는 두 개의 인스턴스가 각각 다른 색상과 모델을 가진다.

인스턴스의 메서드: 인스턴스는 클래스에서 정의한 메서드를 호출 하여 동작을 수행가능
각각의 인스턴스는 자신의 상태에 따라 메서드를 다르게 수행할 수 있음
